Script validates the salinity interpolation framework described in S2. 
The validation adresses two aspects of model perfomance: (i) in-sample raster validations, where salinity values at statnition locations are compared with values sampled directly from the UTM salinity raster generated by the interpolation workflow (i.e., evaluates how closely the rasterized product produces the orignal observations at the sampled points;
and (ii) leave-one-out-cross-validation (LOOCV), where for each stations, the salinity is predicted from all remaining stations using the same along-thalweg interpolation procedure, and the predicted values compared with the observation (i.e., evaluates the predictive skill of the interpolation approach independently of direct point reuse).

Inputs:
- thalweg geometry .gpkg file
- CSVs with salinity observations from each time-step (e.g., monthly). Columns: Point_ID, Latitude, Longitude, Month, Year, Salinity_PSU
- UTM rasters generated by S2_make_salinity_rasters.py

Outputs:
For this study, named as:
00_inputs_check.txt
validation_summary_by_month.csv
residuals_raster_YYYY_MM.csv
residuals_loocv_YYYY_MM.csv
scatter_raster_YYYY_MM.png
scatter_loocv_YYYY_MM.png
residuals_vs_dist_raster_YYYY_MM.png
rmse_comparison_monthly.png

Rationale:
1- Read the thalweg geometry and monthly salinity observations 
2- Project stations coordinates to UTM Zone 28N (EPSG_32628)
3- Compute distance of each stations along the thalweg
4- Sample interpolated salinity values from the intermediate raster at station locations
5- Compared observed and raster-sampled values
6- Perfome LOOCV using 1D interpolation along the thalweg
7- Export residuals tables, summary statistcs, and diagnostics
